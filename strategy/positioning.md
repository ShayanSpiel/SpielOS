# Positioning

**Category:** Session-as-Content Infrastructure (Build-to-Public Pipeline)

> Content is not created. It is extracted from work.

**One line:** The Spiel Engine turns a founder's actual build sessions into publishable content, so they don't need to stop working to create posts.

**Core insight:** Founders fail at content because they must switch from builder mode to creator mode. The Spiel Engine removes that switch entirely. Building = Content generation.

**Core principle:** You should never "go create content." Your work should already contain it.

## The Transformation

**Before:**
Build → stop → think → write → post (rare, inconsistent)

**After:**
Build → session captured → content generated → publish (continuous, invisible overhead)

## Who This Is For (positioning-level)

This is not for beginners. Not for "wantrepreneurs." For someone who has built something real — deep domain expertise, a product, a service, a method — and knows they should be generating leads from it but can't because they're busy building.

## The Hormozi Value Equation

```
Value = (Dream Outcome × Perceived Likelihood) / (Time Delay × Effort & Sacrifice)
```

| Component | Score | Why |
|-----------|-------|------|
| Dream Outcome | ↑↑↑ | Complete inbound system. Not a tool — a transformation. |
| Perceived Likelihood | ↑↑ | I run this system myself. You see the proof daily. 30 days review. |
| Time Delay | ↓↓ | 14 days. Not 6 months. Sample pillars in your voice on day 14. |
| Effort & Sacrifice | ↓↓ | You show up for 1 onboarding session. The system does the work. |
| Price | ↓↓↓ | $990 vs $11,200 value. vs $4k/mo ghostwriter. vs $8k/mo agency. |

## The "What's the Worst That Can Happen" Test

An offer that the user is *proud* to deliver is a good offer. An offer that the user is *anxious* to deliver is a bad offer. Price the offer at a level where the user is *happy* to do the work.

This is a positioning-level principle, but it lives in the offer section. See [[offer §What's the Worst Test]].
