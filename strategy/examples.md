# Example 1

Thread:

Marketing IS the hardest skill in tech.

Product is abundant.

Attention is limited.

I spent ~10 years working on products with millions of users.

Now I'm back at 0.

No audience.

No distribution.

No momentum.

So I'm rebuilding the playbook from scratch.

₁˯

A few days ago I was publishing technical content.

It failed.

No clicks.

No traffic.

No signal.

Then I ran a different experiment.

One post drove almost all new traffic over the last 2 days:

~300 visitors.

6m 58s avg session duration.

Organic social traffic.

People didn't just click.

They stayed.

₂˯

The mistake:

I was creating content.

The shift:

I started attaching to existing attention.

₃˯

Traffic is no longer the problem.

Now the bottleneck is conversion.

On to the next experiment.

Why it works:
It starts with a strong claim, then immediately breaks into real constraints and personal reset. No fluff setup. It shows an experiment, shows failure, then shows a measurable outcome (300 visits, 6:58 duration). The structure feels like a live system being debugged, not advice being taught.

# Example 2

Post:

I tested technical content for a few days.

It failed.

No clicks.

No momentum.

So I changed one thing.

I stopped trying to create attention.

I started entering it.

Now I only build content inside active conversations.

If no one is already talking about it, I don't post.

Why it works:
Single idea. Clear pivot point. The contrast ("failed → changed one variable → new behavior") makes the learning feel real, not theoretical. It reads like an experiment log, not a strategy explanation.

# Example 3

Post X:

GLM 5.2 wasn't the opportunity.

The attention around it was.

Big accounts were already pushing it.

Pricing debates were already happening.

My ICP was already locked in.

So I didn't "create content".

I inserted a system into an existing wave.

Why it works:
It reframes success away from content creation and toward timing + distribution entry. It removes ego ("I didn't create it") and replaces it with system thinking ("I entered the wave").

# Example 4

Post:

~300 visitors from a single post.

No ads.

No audience.

No distribution.

Just placement.

Same content style I used before = nothing.

New placement inside active attention = signal.

That's the only variable that changed.

Why it works:
Extremely concrete. It isolates a single variable change. That's what makes it feel like a real experiment instead of storytelling. The repetition of "no X" builds constraint clarity.

# Example 5

Post:

I don't think most people have a content problem.

They have a distribution problem.

They keep improving writing.

But they never learn where attention actually is.

Writing is not the bottleneck.

Entry point is.

Why it works:
One idea only. No expansion. No justification overload. It states a contrarian position cleanly, then anchors it with a simple mechanism shift (writing vs entry point).

# Example 6 (vulnerability — session-mode confessional)

Post:

I spent two years refusing to post.

I built things. I just didn't talk about them.

The reason wasn't time. It wasn't strategy. It was identity.

Every marketing tool asked me to become a "creator." A "storyteller." A "personal brand."

I built my career on being a builder. Becoming a creator felt like putting on a costume.

So I didn't post.

The thing nobody tells you: builders don't fail at content because of skill. They fail because the tools assume a different identity than theirs.

The fix isn't better content. It's a methodology that treats the work session as the content.

Note: I built this for myself first. The fact that it became a methodology was an accident. But once you see content as a byproduct instead of a separate job, you can't go back.

Why it works:
Confessional opening ("I spent two years refusing to post"). The reader recognizes themselves. The "identity, not skill" reframe is the aha. The "I built this for myself first" close is a self-deprecating note that re-anchors the post in lived experience, not advice.

# Example 7 (paradigm — professional theory)

Post:

The "content is king" era is over.

It's not that content doesn't matter. It's that the unit of value changed.

The old unit: a post. A piece of content. A thing you create.

The new unit: an attention flow. A stream someone enters. A system that routes attention to a destination.

The implications are larger than they sound:

- A great post with bad placement dies.
- A mediocre post in the right flow compounds.
- A "consistent" creator with no flow burns out.

The bottleneck moved. It is no longer in the writing. It is in the routing.

This is why "just post more" stopped working for so many people. The marginal return on a post is now negative. The marginal return on a flow is positive.

Note: the post is not the asset. The flow is.

Why it works:
Paradigm-shifting opener ("The X era is over"). The reader feels a frame shift before they read the body. The "implications are larger than they sound" pivot is the aha. The 3-bullet list proves the new frame without being preachy. The Note: close is a single-sentence crystallizer.

# Example 8 (reader-problem first — perception-shift)

Post:

Remember the "I will organize this note later" pile?

The notes app graveyard. Browser bookmarks from 2015. Physical notebooks with one page filled.

Every builder has one.

The problem is not that you're disorganized. The problem is that the system you use to capture ideas treats them as content to organize, not signals to act on.

So you bookmark it. You tag it. You file it. You never look at it again.

The shift: stop treating ideas as things to keep. Treat them as things to enter into an active system.

A bookmark is a dead end. A session log is alive.

Why it works:
Opens with the reader's problem (the pile), not the author's experience. The rhetorical question and concrete image (graveyard, books from 2015, notebooks with one page) make the reader see their own situation. The "the problem is not X, it's Y" reframe is the aha. The close contrasts two mental models (bookmark vs session log) without telling the reader what to do.

# How to pick an example for a draft

Match the Belief → Point shape in the brief to the rhetorical shape of the example:

| Belief → Point shape | Match to example |
|---|---|
| "I failed because I did X wrong. The shift: do Y instead." | Example 1, 2, 3, 4 (experiment log) |
| "Common belief X is wrong. The real pattern is Y." | Example 5, 7 (contrarian / paradigm) |
| "I avoided / struggled with X. The lesson: Y." | Example 6 (vulnerability) |
| "You have a problem you don't realize you have. The shift: Y." | Example 8 (reader-problem first) |

The Strategist's `example_pattern` field in the brief names which example to mirror rhythmically. The Writer reads that example before drafting.
